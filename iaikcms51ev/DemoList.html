IAIK-CMS Demo List

Cryptographic Message Syntax / S/MIME Version 3

This page contains a list of all demos that are included in the IAIK-CMS distribution. According to their functionality we can distinguish between CMS demos (packages demo.cms.*) and S/MIME demos (package demo.smime.*).

The following jar files are required by all demos:

  • iaik_jce.jar (or iaik_jce_full.jar; required for running the ESDH demos included in class CMSEnvelopedData)
  • iaik_cms.jar
  • iaik_cms_demo.jar
Additional IAIK libraries that may be required by particular demos are listed in the tables below. If you are using a JDK version > 1.4 you will have to use the signed version of iaik_jce_(full).jar If you are using a JDK 1.1.x version, you also must have jdk11x_update.jar (included in the IAIK-JCE distribution) in your classpath (please see installation guidelines of IAIK-JCE).

S/MIME Demos

(contained in packages demo.smime.*): demonstrate how to use the S/MIME library for creating, signing, encrypting, sending,... and receiving, parsing, verifying, decrypting S/MIME messages. In addition to the iaik_jce_(full).jar, iaik_cms.jar and iaik_cms_demo.jar libraries you need the JavaBeansTM Activation Framework (JAF) and the JavaTM Mail API for all the following demos. You can get the JavaBeansTM Activation Framework (activation.jar) from http://java.sun.com/products/javabeans/jaf/index.jsp, and the JavaMailTM API (mail.jar; pop3.jar, if required for POP3 access) from http://java.sun.com/products/javamail/. Note that, if you are using JDK 6 or later, the JavaBeansTM Activation Framework is already included in the JDK; therefore in this case you do not need to put activation.jar into your classpath. You also should not use version 1.3.3 of the JavaMailTM API because of a bug in its Base64 decoding routine. Version 1.4 of the JavaMailTM API and version 1.1 of the JavaBeansTM Activation Framework can be only used with JDK 1.4 or later.
